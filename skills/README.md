# AI辅助产品经理工作流使用指南

欢迎使用这套专为新手产品经理（或希望系统化学习产品规划的人）打造的AI辅助工作流！

传统的文档编写往往让人无从下手，本工作流打破了常规的“先想清楚再写文档”的模式，采用**“启发式对话引导 -> 详细的初版 PRD -> 原型视觉化验证 -> 逻辑与细节补全”**的敏捷迭代方式，帮助你一步步理清思路。

## 📁 目录包含哪些文件？

1. **`pm_workflow_definition.md`**：核心工作流指令文件。它告诉AI应该如何引导你、如何评估你的需求、以及按照什么步骤产出内容。
2. **`prd_template.md`**：PRD（产品需求文档）的标准结构模板。AI最终会按照这个结构为你生成专业的HTML文档。
3. **`README.md`**：也就是本文件，你的使用说明书。

---

## 🚀 如何开始使用？

### 第一步：启动工作流
将 `pm_workflow_definition.md` 的内容发送给你的AI助手，并附上你最原始、最模糊的想法。
> **示例**：
> "请按照这个工作流帮我梳理需求：我想做一个给装修工人用的打卡小程序，能拍照就行。"

### 第二步：跟随AI的引导（需求采集与确认）
不要害怕你的需求说不清楚。AI会主动评估你缺失了什么信息（比如：目标用户是谁？打卡需要定位吗？），并向你提问。
**你只需要像聊天一样回答AI的问题即可。** 经过几轮沟通，AI会给你一个结构化的需求确认清单。

### 第三步：项目初始化与详细的第一版初步PRD
需求确认后，AI会主动帮你建立一个清晰、规范的项目文件夹结构（包括存放PRD、原型、流程图的文件夹），告别文件存放混乱。
接着，AI会为你生成一份**非常详细的初版 HTML PRD**（包含项目信息、背景目标、有代入感的用户旅程图，以及详细的核心操作主线和业务规则）。
这个时候的文档不是一个空框架，而是具备基本业务逻辑闭环的文档，供你确认整体方向和基础逻辑。你可以提出修改意见。

### 第四步：进入“原型视觉化”阶段（关键！）
对于新手来说，直接写完全部逻辑很困难。所以我们的工作流会在这里**转而先做高保真原型**。
AI会根据初版PRD，为你生成单页面的HTML原型（采用Tailwind CSS）。
> **为什么这样做？** 看着实际的画面，你更容易发现：“哎呀，这里好像少了一个取消按钮”、“这个步骤逻辑不对”。通过修改视觉原型，倒逼你完善业务逻辑，并同步更新 PRD。

### 第五步：绘制流程图与最终版 PRD
原型跑通后，AI会为你绘制核心操作主线的 Mermaid 流程图。
最后，AI会生成最终版的HTML格式的PRD。
最酷的是，AI会将刚才做好的原型，按照功能点“切片”或以交互式的 `iframe` 嵌入到PRD的对应表格中，实现“左边是文档，右边（或旁边）是可操作的原型”。

## 💡 推荐搭配的强大工具 (Skills)

为了让原型的产出达到极其专业和美观的水平，强烈建议你在你的 AI IDE (如 Trae) 中安装并使用前端设计专家级指令集：**Impeccable Skills**。

### 什么是 Impeccable？
它是基于基础前端技能的加强版，包含了一整套设计语言和规范（如 `/typeset` 修复排版、`/arrange` 修复布局等）。它能让 AI 生成的 HTML 原型告别“简陋感”，拥有现代、高级、极其精致的视觉体验。

### 如何安装与使用？
1. **安装配置**：你可以访问官方指南 [https://impeccable.style/](https://impeccable.style/)，使用命令 `npx skills add pbakaus/impeccable` 在你的项目根目录下自动安装。
2. **初始化设计规范**：安装后，在对话框输入 `/teach-impeccable`，它会收集你的设计偏好并生成配置文件，后续所有的设计都会遵循这个规范。
3. **生成原型**：当工作流走到**第三步（产出原型）**时，AI 会调用 `frontend-design` 等技能，并结合 Impeccable 的规范为你生成高保真原型。
4. 这样生成的原型不仅可以直接演示，其 UI 规范也能直接作为后续开发的参考。

## 🌟 进阶提示：为什么我们要加入“用户旅程图”？
在最新的 PRD 模板中，我们特别增加了**“用户使用场景/用户旅程图”**的章节。
新手极易陷入“为了做功能而做功能”的陷阱。在画原型和写逻辑之前，强迫自己写出**“用户在什么场景下，遇到了什么问题，通过我们的产品经历了哪几个步骤，最终获得了什么结果”**，能极大地提升产品的可用性和同理心。
