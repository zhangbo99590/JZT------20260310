# 搜索功能需求文档与规范

## 1. 概述
本文档旨在规范政策搜索结果列表的展示逻辑、交互行为及技术要求，确保搜索功能准确、高效、用户体验良好。针对当前存在的“筛选无结果”及“匹配逻辑不精确”问题进行了针对性优化。

## 2. 搜索逻辑与匹配规则

### 2.1 关键词匹配
*   **输入处理**：对用户输入的关键词进行去空、转小写处理。
*   **匹配字段**：
    *   政策标题 (`title`)
    *   关键词标签 (`keywords`)
    *   发布单位 (`publishOrg`)
    *   适用行业 (`applicableIndustries`)
*   **匹配方式**：采用模糊匹配 (`includes`)，只要任一字段包含关键词即视为匹配。

### 2.2 筛选条件匹配
*   **区域筛选 (`districts`)**：
    *   支持多选。
    *   逻辑关系：OR（或）。即政策区域只要包含任一选中的区域即可。
    *   特殊处理：支持“区/市”后缀的模糊匹配（如筛选“朝阳区”，可匹配“朝阳”）。
*   **行业筛选 (`industries`)**：
    *   支持多选。
    *   逻辑关系：OR（或）。
    *   **复杂映射逻辑**：
        *   筛选值可能为复合字符串（如“城乡建设、环境保护 / 节能与资源综合利用”）。
        *   **匹配规则**：将筛选值按 `/` 分割，取最后一部分作为核心关键词进行匹配。例如，“城乡建设、环境保护 / 节能与资源综合利用”应提取“节能与资源综合利用”或“节能”、“资源”等核心词。
        *   **双向匹配**：
            1.  政策行业包含筛选关键词。
            2.  筛选关键词包含政策行业（如筛选“节能与资源综合利用”可匹配“节能环保”）。
*   **级别筛选 (`levels`)**：精确匹配（国家级、省级、市级等）。
*   **类别筛选 (`categories`)**：精确匹配。

### 2.3 排序规则
*   **默认排序**：按发布时间倒序（最新的在前）。
*   **相关度排序**：关键词匹配度高的在前（需后端或算法支持，前端暂按时间）。

## 3. 结果列表展示规范

### 3.1 列表项（卡片）结构
每个搜索结果卡片应包含以下信息：
1.  **标题**：高亮显示匹配的关键词。
2.  **标签行**：
    *   政策级别（如“国家级”）：红色背景。
    *   政策类别（如“立项公示”）：蓝色背景。
    *   其他特征标签。
3.  **核心指标**：
    *   **资助总额** / **最高奖励**：突出显示数字。
    *   **已获批企业**：显示数量。
4.  **元数据**：
    *   发布日期（格式：YYYY-MM-DD）。
    *   发布单位。
    *   适用行业。

### 3.2 空状态处理
*   **触发条件**：搜索结果数组长度为 0。
*   **展示内容**：
    *   友好的提示文案：“未找到符合条件的政策”。
    *   建议操作：“建议您调整筛选条件或更换关键词”。
    *   **重置按钮**：提供“重置筛选”按钮，点击后清空所有筛选条件并重新搜索。

### 3.3 加载状态
*   **触发条件**：搜索请求发起至结果返回期间。
*   **展示内容**：显示 Loading 骨架屏或旋转图标，遮罩在结果区域。

## 4. 技术要求

### 4.1 响应式布局
*   支持桌面端（1200px+）、平板（768px-1199px）和移动端（<768px）。
*   移动端应适配卡片布局，适当隐藏非核心信息。

### 4.2 性能优化
*   **防抖**：搜索输入框应有 300ms-500ms 的防抖处理。
*   **分页**：前端假分页或后端真分页，每页默认 10 条。
*   **缓存**：对相同的查询条件结果进行短时缓存（如使用 `useMemo` 或 React Query）。

### 4.3 错误处理
*   **网络错误**：显示“网络连接失败，请稍后重试”并提供重试按钮。
*   **数据异常**：对缺失字段进行兜底显示（如“未公开”），避免页面崩溃。

## 6. 待修复问题分析与方案
**当前问题**：筛选“城乡建设、环境保护 / 节能与资源综合利用”时，无法匹配到“节能环保”类的政策。
**原因**：前端筛选逻辑使用的是简单的 `includes`，且筛选值过长，导致无法匹配到简短的行业标签。
**解决方案**：优化 `searchEnhancedPolicies` 中的行业匹配逻辑，支持分割筛选值并进行模糊匹配。
