## 1. 首页与工作台 (Dashboard)

### 1.1 企业画像与智能推荐
- **功能描述**：
  - 企业用户登录后，需引导完善**企业画像标签**（如行业、规模、资质等）。
  - 系统基于企业画像标签和资质信息，在首页智能推荐匹配的政策、服务和资讯。
- **业务流程**：
  1. 用户登录。
  2. 检测企业画像完整度。
  3. 若未完善，引导至`企业管理 -> 编辑用户画像`页面。
  4. 完善后，首页推荐模块实时更新匹配内容。

### 1.2 企业认证与邀请机制
- **新增入口**：在系统显眼位置新增“企业认证”入口。
- **认证流程**：
  1. 用户提交企业认证资料。
  2. 认证通过后，该用户自动成为**超级管理员**。
  3. 系统生成唯一的**企业邀请码**。
- **邀请功能**：
  - 超级管理员可将邀请码分享给员工。
  - 员工注册/登录时输入邀请码，自动加入该企业组织，获得对应权限。

### 1.3 登录与权限优化
- **登录方式**：
  - 支持**密码登录**。
  - 支持**手机验证码登录**。
- **注册流程**：完善注册流程，增加必要的校验和引导。
- **权限管理**：优化权限控制逻辑，确保不同角色（超级管理员、管理员、普通员工）的权限边界清晰。

### 1.4 UI/UX 优化 (待定)
- **最近活动**：重新设计展示规则，确保信息更具时效性和相关性。
- **重要提醒**：优化重要提醒页面展示，突出紧急待办事项。

## 2. 政策中心 (Policy Center)

### 2.1 搜索结果页优化
- **页面精简**：搜索结果页**仅保留政策文件列表**。
- **模块移除**：删除“相关项目推荐”和“扶持企业推荐”板块，聚焦政策本身。

### 2.2 未收录企业搜索体验优化 (核心)
- **场景**：当用户搜索数据库中不存在的企业名称时。
- **交互逻辑**：
  1. **避免空白页**：严禁直接显示“无数据”或空白页面。
  2. **状态告知**：顶部清晰提示“未查询到【XXX企业】的平台记录”。
  3. **价值传递**：说明完善信息的好处（如“精准政策匹配”、“补贴申报辅助”）。
  4. **引导操作**：
     - **主按钮（蓝色）**：“完善企业信息” -> 跳转至信息补全页。
     - **次按钮（灰色）**：“仅本次临时匹配” -> 允许用户输入临时数据进行一次性测算。
  5. **进度展示**：同步展示“基本信息 / 知识产权 / 经营数据 / 已获批政策”四个模块的完成进度（初始为 0%）。
- **闭环流程**：搜索校验 → 空状态引导 → 信息完善 → 政策匹配 → 数据留存。

### 2.3 申报管理优化
- **流程简化**：取消“申报管理”的概览首页。
- **交互调整**：点击“申报管理”菜单或入口，**直接进入申报流程/申报列表页**，减少中间环节。

## 4. 产业管理 (Industry Management)

### 4.1 业务大厅 (Service Hall) - 供给侧
- **核心定位**：企业对外展示“我能提供什么服务/产品”。
- **发布内容**：
  - **专业服务**：食品检测、安全认证、研发外包、设备租赁等。
  - **企业能力**：技术服务、代工生产等。
- **界面特征**：
  - **条目**：企业名称 + 发布时间。
  - **描述**：“业务供给：提供 XX 专业服务 / 产品”。
  - **标签**：业务类型、地域。
  - **操作**：立即对接、收藏、对比。
- **隐私保护**：部分敏感信息需进行**打码展示**，对接后可见。

### 4.2 采购大厅 (Procurement Hall) - 需求侧
- **核心定位**：企业对外发布“我需要什么服务/产品”。
- **发布内容**：
  - **设备采购**：数控机床、生产设备等。
  - **服务采购**：数字化改造、园区管理方案等。
  - **物资采购**：原材料、办公用品等。
- **用户角色**：
  - **发布方（需求方）**：发布需求，编辑草稿，取消需求，查看进度。
  - **对接方（供给方）**：浏览需求，立即对接，收藏，对比。
  - **运营方**：审核需求，分配跟进人，记录日志。
- **界面特征**：
  - **描述**：“采购需求：需要 XX 服务 / 产品”。
  - **标签**：采购品类、地域、预算区间。
