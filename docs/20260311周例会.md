# 产业管理
 业务大厅：打码展示

一、业务大厅（供给侧场景）

核心定位：企业对外展示「我能提供什么服务 / 产品」，是业务供给的发布与撮合平台。

发布内容：以「业务供给」为核心，比如：

专业服务：食品检测、安全认证、医疗器械研发外包、废水处理设备租赁等

企业能力：技术服务、设备租赁、代工生产等

用户角色：

发布方：有服务 / 产品能力的企业（供给方）

对接方：有采购 / 合作需求的企业（需求方）

界面特征：

条目标题：企业名称 + 发布时间

内容描述：「业务供给：提供 XX 专业服务 / 产品」

标签：业务类型（如「食品检测」「医疗器械」）、地域

操作：「立即对接」（需求方联系供给方）、收藏、对比

采购大厅：思考采购大厅的展现方式

核心定位：企业对外发布「我需要什么服务 / 产品」，是采购需求的发布与撮合平台。

发布内容：以「采购需求」为核心，比如：

+ 设备采购：高端五轴联动数控机床、锂电池湿法隔膜等
+ 服务采购：智慧园区综合管理平台解决方案、生产线数字化改造服务等
+ 物资采购：生产原材料、办公设备等

用户角色：

+ 发布方：有采购 / 合作需求的企业（需求方）
+ 对接方：有服务 / 产品能力的企业（供给方）
+ 运营方：平台业务人员（负责需求审核、跟进撮合）

界面特征：

+ 条目标题：企业名称 + 发布时间
+ 内容描述：「采购需求：需要 XX 服务 / 产品」
+ 标签：采购品类（如「数控机床」「锂电池」）、地域、预算区间
+ 操作：
    - 需求方：编辑草稿、取消需求、查看跟进进度
    - 供给方：「立即对接」（联系需求方）、收藏、对比
    - 运营方：审核需求、分配跟进人、记录跟进日志
