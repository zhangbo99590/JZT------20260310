# 政策搜索与智能匹配功能差异说明文档

## 1. 功能定义与业务目标

| 维度 | 政策搜索 (Policy Search) | 智能匹配 (Policy Matching) |
| :--- | :--- | :--- |
| **核心定义** | 基于用户输入的主动查询行为，通过关键词、标签等手段，从海量政策库中召回相关政策信息。 | 基于企业全维画像与政策规则库的自动化比对，计算企业与政策的适配程度，推送高精准度的申报建议。 |
| **业务目标** | **侧重召回广度 (Recall)**。帮助用户快速定位特定政策文件，支持模糊查询，解决“找政策难”的问题。 | **侧重精准度 (Precision)**。解决“读不懂、报不准”的痛点，通过算法量化申报成功率，辅助决策。 |
| **用户场景** | 用户已知部分关键词（如“高企认定”），需要查找具体文件或了解相关政策动态。 | 用户不知晓有哪些政策适合自己，希望系统根据企业情况主动推荐最优申报方案。 |

## 2. 输入数据格式与预处理要求

| 维度 | 政策搜索 | 智能匹配 |
| :--- | :--- | :--- |
| **输入数据** | - **显式输入**：搜索关键词（Keywords）、筛选条件（地区、部门、发布时间）。<br>- **隐式输入**：用户浏览历史（用于个性化排序）。 | - **企业画像**：工商注册信息、知识产权、财务数据（营收、利润、研发投入）、人员结构。<br>- **项目结构化字段**：技术领域、投资额、建设周期等。 |
| **预处理** | - **分词与去停用词**：对搜索词进行分词处理（如“北京市/高新技术/认定”）。<br>- **同义词扩展**：建立同义词库（如“专精特新”=“小巨人”）。 | - **数据清洗**：企业数据的归一化（如金额统一单位为万元）。<br>- **特征工程**：提取关键特征（如“近三年研发投入占比”、“是否上市”）。 |

## 3. 算法策略与模型差异

| 维度 | 政策搜索 | 智能匹配 |
| :--- | :--- | :--- |
| **核心算法** | **倒排索引 + BM25**。基于文本相关性的检索模型，辅以时间衰减因子。 | **向量化 + 规则评分 + 机器学习排序 (LTR)**。<br>1. **粗排**：基于硬性门槛（地区、行业、注册时间）快速过滤。<br>2. **精排**：计算特征向量相似度，结合规则引擎（Rule Engine）进行评分。 |
| **技术栈** | Elasticsearch / Solr | Python (Scikit-learn/TensorFlow) + 规则引擎 (Drools/EasyRules) |

## 4. 输出结果排序逻辑与阈值策略

| 维度 | 政策搜索 | 智能匹配 |
| :--- | :--- | :--- |
| **排序逻辑** | **按相关性得分降序**。<br>综合考虑：关键词匹配度 > 发布时间 > 政策热度（浏览量）。 | **按匹配度百分比 + 政策优先级双维度排序**。<br>优先展示匹配度 > 80% 且 补贴金额高/截止日期近 的政策。 |
| **阈值策略** | 较低的阈值，尽可能展示更多结果（Top N）。 | 严格的阈值（如匹配度 < 60% 不推荐），宁缺毋滥。 |

## 5. 性能基准

| 指标 | 政策搜索 | 智能匹配 |
| :--- | :--- | :--- |
| **QPS (Queries Per Second)** | ≥ 800 | ≥ 50 (计算密集型) |
| **响应时间 (P99)** | ≤ 200 ms | ≤ 500 ms (并发 100 时) |
| **并发能力** | 高并发，支持水平扩展。 | 中等并发，依赖计算资源。 |

## 6. 异常与边界处理

| 场景 | 政策搜索 | 智能匹配 |
| :--- | :--- | :--- |
| **无结果/少结果** | - **兜底策略**：推荐热门政策或相关领域的通用政策。<br>- **拼写纠错**：提示“您是不是要找...”。 | - **提示优化**：明确告知不匹配的原因（如“缺少研发数据”、“注册地不符”）。<br>- **引导完善**：引导用户补充企业信息以提高匹配度。 |
| **数据异常** | 能够处理特殊字符、超长搜索词。 | 需处理字段缺失（Missing Values）、政策失效（过期政策自动降权或隐藏）、版本回退。 |

## 7. 可观测性要求

为确保系统稳定运行及持续优化，两模块均需满足以下可观测性指标：

*   **Trace ID**：全链路追踪 ID，贯穿请求始末。
*   **核心指标**：
    *   **搜索**：搜索耗时、零结果率、CTR (点击率)、Top10 搜索词。
    *   **匹配**：匹配耗时、匹配成功率、Top1 推荐政策 ID、用户采纳率。
*   **日志记录**：详细记录输入参数、算法得分中间值、最终输出结果。
